# 「特定メール → スプレッドシート → Slack通知」自動化システム

## このドキュメントの使い方

下記の **「AIへの依頼プロンプト」** セクションをまるごとコピーして、ChatGPT・Claude・Gemini などに貼り付けてください。AIが Google Apps Script のコードと導入手順を出力します。出力結果をそのまま Google Apps Script に貼って動かす、という流れです。

プログラミング経験は不要です。Google アカウントと Slack ワークスペースの管理権限さえあれば動かせます。

---

## 事前に用意するもの

1. **通知対象メールが届いている Gmail アカウント**
2. **Slack ワークスペースの管理権限**（Incoming Webhook を発行するため）
3. **新規 Google スプレッドシート1枚**

```
あなたは Google Apps Script の専門家です。以下の自動化システムを作ってください。

# 目的
既存の Gmail 受信箱に「特定の条件にマッチするメール」が届いたら、
1. Google スプレッドシートに自動で1行追記する
2. Slack グループチャットに Incoming Webhook で通知を送る
3. スプレッドシート上で対応状況（ステータス・担当者・期限・メモ）を管理できる

# 重要な制約
- 新しい Gmail アカウントは作らない。既存の受信箱の中で、Gmail フィルターでラベル付けされたメールだけを対象にする
- フィルター条件にマッチしないメール（個人メール・社内メール等）には一切触らない
- Webhook URL や シートID はコードに直書きせず、Apps Script の「スクリプトプロパティ」から読み込む
- Gmail の権限は「ラベル付きメールの読み取り」と「ラベル変更」のみ。送信・削除はしない

# スプレッドシート構造
シート名は `inquiries`。以下13列で、行1はヘッダ。

| 列 | 名称 | 入力 | 内容 |
|---|---|---|---|
| A | ID | 自動 | 連番 |
| B | 受信日時 | 自動 | Gmailの受信日時 |
| C | 送信元 | 自動 | From |
| D | 件名 | 自動 | Subject |
| E | 本文プレビュー | 自動 | 先頭500文字、改行は空白に置換 |
| F | Gmailリンク | 自動 | https://mail.google.com/mail/u/0/#inbox/{threadId} |
| G | ステータス | 手動 | プルダウン: 未対応/対応中/完了/保留 |
| H | 担当者 | 手動 | プルダウン（候補は別シート `_config` の A列から取得） |
| I | 対応期限 | 手動 | 日付。期限超過は赤、当日は黄に色付け |
| J | 対応メモ | 手動 | 自由記述 |
| K | 完了日時 | 自動 | ステータスが「完了」に変わったら自動入力 |
| L | Slack通知済 | 自動 | true/false。Webhook成功でtrue、失敗ならfalse(次回再送) |
| M | Gmail Message ID | 自動 | 重複処理防止用 |

# 動作仕様
## 0. UX 強化（必須）
- onOpen: スプレッドシート起動時にカスタムメニュー「問い合わせパイプライン」を追加。「今すぐメール取得」「Slack通知テスト」「ヘルスチェック」「セットアップ実行（初回のみ）」のメニュー項目を持つ
- healthCheck 関数: スクリプトプロパティ／シート構造／Gmailラベル／トリガー／Slack Webhook疎通を順に確認し、結果を改行区切りで返す。各項目に ✓ / ✗ マークを付ける
- testNotification 関数: メール送信なしでSlackにテスト通知を送る。Webhook URL変更時の確認用

## 1. fetchAndSync 関数（5分毎の時限トリガー）
- Gmail ラベル「問い合わせ-未処理」が付いたスレッドを取得
- 各スレッドの最新メッセージについて：
  - M列の Message ID と照合し、未処理なら新規行として A〜F, M列に追記、G列に「未対応」、L列にfalse
  - Slack に Incoming Webhook で通知を送信。成功なら L列を true に更新
  - Gmail のラベルを「問い合わせ-未処理」→「問い合わせ-処理済」に付け替え
- 処理の最後に L列が false のままの行を再度 Slack に送信（リトライ）

## 2. onSheetEdit 関数（インストール型 onEdit トリガー）
- `inquiries` シートの G列（ステータス）が「完了」に変わったら、K列に現在日時を自動入力（既に入っていれば上書きしない）

## 3. setup 関数（初回手動実行）
- `inquiries` シートと `_config` シート（隠し）を作成（無ければ）
- ヘッダ行を太字＋背景色で書き込み、行1を固定
- G列にステータスのプルダウンを設定
- H列に `_config` A2:A の範囲を参照する担当者プルダウンを設定
- G列の値ごとに条件付き書式（未対応=薄赤、対応中=薄黄、完了=薄グレー、保留=薄グレー）
- I列の条件付き書式（期限超過＆未完了＆未保留=赤、当日＆未完了＆未保留=黄）
- Gmail ラベル「問い合わせ-未処理」「問い合わせ-処理済」を作成（無ければ）
- fetchAndSync の時限トリガー（5分毎）と onSheetEdit のスプレッドシート編集トリガーを登録（既存があれば削除して再登録）

## 4. Slack 通知の内容（Block Kit形式）
- header: 「[<クライアント名>] 新規問い合わせ」
- section: 件名と From を fields で並べる
- section: 本文プレビュー150文字を引用ブロック（>）で
- actions: 「Gmailで開く」「シートで管理」の2ボタン
  - シートで管理のURL: https://docs.google.com/spreadsheets/d/{SHEET_ID}/edit#gid={GID}&range=A{row}

# スクリプトプロパティ
以下の3つを使う：
- SHEET_ID: スプレッドシートID（必須）
- SLACK_WEBHOOK_URL: Slack Incoming Webhook URL（必須）
- ADMIN_EMAIL: エラー通知先メールアドレス（任意）

# エラーハンドリング
- 各スレッド処理は try/catch で囲み、失敗してもループは続行
- 例外は console.error にログ出力
- ADMIN_EMAIL が設定されていればエラー内容をメール送信
- Slack 通知失敗時もシート追記は完了させ、L列のfalseフラグで次回リトライ

# 出力してほしいもの
1. 上記すべてを満たす Apps Script コード（単一ファイル `Code.gs`、そのままコピペで動くもの）
2. セットアップ手順（Slack Webhook 発行 → スプレッドシート作成 → Apps Script 貼付 → スクリプトプロパティ設定 → setup() 実行 → Gmail フィルター作成 → 動作確認）
3. 動作確認の手順（4〜5パターン、フィルター除外テストを含む）
4. トラブルシューティング（通知が来ない・行が追加されない・重複が出る・個人メールが拾われた・緊急停止したい）
5. カスタムメニューの使い方（運用者がスプレッドシートから「今すぐメール取得」「Slack通知テスト」「ヘルスチェック」を呼び出せること）

# このシステムを使う環境
- クライアント名: <例: FIRE_KIDS>
- 通知対象メールの想定: <例: お問い合わせフォーム送信通知 / EC注文通知>
- Slack 通知先チャンネル: <例: #firekids-inquiries>
- 想定運用人数: <例: 3〜5名>

日本語でコメントとセットアップ手順を書いてください。コード内に Webhook URL や SHEET_ID をハードコードしないでください。
```

## このプロンプトに含めた工夫

AIが迷わず動くものを作れるよう、以下を盛り込んでいます：

- **完全な列定義**：13列すべての名称・型・自動/手動の区別を明記
- **トリガー仕様**：3つの関数の動作を具体的に記述
- **失敗時の挙動**：リトライ・重複防止・エラー通知まで指定
- **セキュリティ要件**：スクリプトプロパティ利用、Gmail権限の限定
- **出力フォーマット**：コード＋セットアップ手順＋テスト＋トラブルシューティングを明示
- **環境変数**：山かっこ部分だけ書き換えれば他クライアントにも転用可能

---

## 動作させるまでの流れ

1. **AI に上記プロンプトを投げる** → コード＋手順書が返ってくる
2. **Slack で Incoming Webhook を発行** → URL を控える（[Slack Apps](https://slack.com/apps) から「Incoming Webhooks」を追加）
3. **新規スプレッドシートを作成** → URLからシートIDを控える（`spreadsheets/d/<ここ>/edit`）
4. **スプレッドシートから Apps Script を開く**（拡張機能 → Apps Script）
5. **AI が出力したコードを Code.gs に貼り付け**
6. **スクリプトプロパティを設定**（プロジェクトの設定 → スクリプトプロパティ）
   - `SHEET_ID`、`SLACK_WEBHOOK_URL`、`ADMIN_EMAIL`
7. **`setup()` を一度実行**（権限承認ダイアログで「許可」）
8. **Gmail フィルターを作成**：通知対象メールに `問い合わせ-未処理` ラベルを付ける条件を設定
9. **テストメールで動作確認**

ここまでで 30〜45 分。

---

## 別クライアント／別用途への転用

プロンプト末尾の山かっこ部分（クライアント名・対象メール想定・Slackチャンネル・運用人数）を書き換えれば、同じ仕組みを別の用途に展開できます。例：

- 採用応募メール → 採用管理シート＋採用チャンネル通知
- 取引先からの請求書メール → 経理処理シート＋経理チャンネル通知
- 特定キーワードを含むメール → 監視シート＋アラートチャンネル通知

スプレッドシート列構成（ステータスや担当者）を変えたい場合は、プロンプトの「スプレッドシート構造」表を書き換えてからAIに投げてください。
