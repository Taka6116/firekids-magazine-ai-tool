# [FIRE_KIDS] 問い合わせ管理パイプライン 導入手順書

特定のメール受信を起点に、スプレッドシートへの自動記録と Slack 通知を行う仕組みです。
既存の Gmail 受信箱内で「特定条件にマッチしたメール」のみを対象とするため、新規の Gmail アカウントは不要です。

- 想定構築時間: 30〜45分
- 必要なもの:
  - 通知対象メールを受信している Gmail アカウント
  - 通知先 Slack ワークスペースの管理権限（Incoming Webhook 発行のため）
  - 新規スプレッドシート1枚

---

## 1. システムの全体像

```
[通知対象メール（フォーム/EC/SNS等）]
              │
              ▼
[既存のGmail受信箱]
              │
              │ Gmailフィルター → ラベル「問い合わせ-未処理」付与
              ▼
[Google Apps Script（5分毎に起動）]
              │
              ├─→ [スプレッドシート（受信ログ＋タスク管理）]
              │
              └─→ [Slack Incoming Webhook → グループチャットに通知]
```

### できること
- 特定のメールが届いたら自動的にスプレッドシートに行が追加される
- 同時に Slack グループチャットに通知が飛ぶ
- スプレッドシート上でステータス・担当者・期限・メモを管理できる
- ステータスを「完了」にすると完了日時が自動入力される

### やらないこと（スコープ外）
- Slack からのステータス変更（一方向通知のみ）
- 自動返信メール
- AI による問い合わせ内容の自動分類
- 担当者の自動アサイン

---

## 2. スプレッドシート構造

GAS の `setup()` 実行時に自動で作成されます。

| 列 | 名称 | 入力 | 説明 |
|---|---|---|---|
| A | ID | 自動 | 連番 |
| B | 受信日時 | 自動 | Gmail受信日時 |
| C | 送信元 | 自動 | From |
| D | 件名 | 自動 | Subject |
| E | 本文プレビュー | 自動 | 先頭500文字 |
| F | Gmailリンク | 自動 | スレッドURL |
| **G** | **ステータス** | **手動** | 未対応/対応中/完了/保留 |
| **H** | **担当者** | **手動** | プルダウン |
| **I** | **対応期限** | **手動** | 期限超過は赤、当日は黄 |
| **J** | **対応メモ** | **手動** | 自由記述 |
| K | 完了日時 | 自動 | ステータス＝完了で自動入力 |
| L | Slack通知済 | 自動 | 通知失敗時は次回再送 |
| M | Gmail Message ID | 自動 | 重複防止用 |

## 3. セットアップ手順

### ステップ 1: Slack Incoming Webhook を発行

1. Slack ワークスペースで [Incoming Webhooks アプリ](https://slack.com/apps) を追加
2. 通知先チャンネルを選択
3. 発行された **Webhook URL** を控える（後で使用）
   - 形式: `https://hooks.slack.com/services/XXX/YYY/ZZZ`

### ステップ 2: スプレッドシートを新規作成

1. Google ドライブで新規スプレッドシートを作成
2. 任意の名前を付ける（例: `[FIRE_KIDS] 問い合わせ管理`）
3. URL からシートIDを控える
   - URL の `https://docs.google.com/spreadsheets/d/<ここがシートID>/edit` の部分

### ステップ 3: Apps Script プロジェクトを作成

1. 上で作ったスプレッドシートを開く
2. メニュー「拡張機能」→「Apps Script」をクリック
3. 開いたエディタの `Code.gs` の中身をすべて削除
4. 本書末尾の **「付録: GAS スクリプト全文」** をコピーして貼り付け
5. プロジェクト名を任意に変更（例: `firekids-inquiry-pipeline`）
6. 保存（Ctrl+S / Cmd+S）

### ステップ 4: スクリプトプロパティを設定

1. Apps Script エディタ左の歯車アイコン「プロジェクトの設定」を開く
2. 一番下の「スクリプト プロパティ」セクションで「スクリプト プロパティを追加」をクリック
3. 以下の3つを設定:

| プロパティ名 | 値 |
|---|---|
| `SHEET_ID` | ステップ2で控えたスプレッドシートID |
| `SLACK_WEBHOOK_URL` | ステップ1で発行したWebhook URL |
| `ADMIN_EMAIL` | エラー通知を受け取るメールアドレス（任意） |

4. 「スクリプト プロパティを保存」をクリック

### ステップ 5: `setup()` を一度だけ実行

1. Apps Script エディタの上部、関数選択プルダウンから `setup` を選ぶ
2. 「実行」ボタンをクリック
3. **初回は権限承認ダイアログが出る**:
   - 「権限を確認」をクリック
   - 自分のGoogleアカウントを選択
   - 「詳細」→「（プロジェクト名）に移動」をクリック
   - 「許可」をクリック
4. 実行ログに「セットアップ完了」と表示されればOK

この時点で自動的に以下がセットアップされます:
- スプレッドシートに `inquiries` シート + `_config` シート（隠し）が作成される
- ステータス・担当者のプルダウン、条件付き書式が適用される
- Gmail に `問い合わせ-未処理` / `問い合わせ-処理済` ラベルが作成される
- 5分毎の自動実行トリガー、編集時トリガーが登録される

### ステップ 6: 担当者プルダウン候補を登録

1. スプレッドシートに戻る
2. 「表示」メニュー →「非表示のシート」→「_config」を開く
3. A2 セル以降に担当者の名前を入力（例: 佐藤、田中、鈴木）
4. H列のプルダウンに反映されることを確認

### ステップ 7: Gmail フィルターを設定（最重要）

通知対象とするメールの条件を Gmail フィルターで定義します。

1. Gmail を開き、検索バー右の「検索オプションを表示」アイコンをクリック
2. 通知対象としたい条件を入力:
   - **From**: 例 `noreply@firekids.jp` や `*@base.in`
   - **件名**: 例 `お問い合わせ` `新規注文` `ご注文ありがとうございます`
   - 必要に応じて本文キーワード
3. 「フィルタを作成」をクリック
4. 「ラベルを付ける」にチェック → `問い合わせ-未処理` を選択
5. 「フィルタを作成」をクリック
6. 必要に応じて複数のフィルターを作成（OR条件で増やしていく）

**設計の指針**:
- 個人メール・社内メールを誤って拾わないよう、From条件は必ず含める
- 初期は条件を広めに設定し、運用しながらチューニングする
- 拾いすぎ（過剰検知）よりも、漏れに寄せる

## 4. 動作確認

### テスト 1: 正常系
1. フィルター条件にマッチするテストメールを自分宛に送る
2. 5分以内に Gmail で `問い合わせ-未処理` ラベルが付くことを確認
3. さらに数分後、スプレッドシートに行が追加され、Slack に通知が届くことを確認
4. メールのラベルが `問い合わせ-処理済` に変わっていることを確認

### テスト 2: フィルター除外
1. フィルター条件にマッチしないメール（個人メール等）を送る
2. ラベルが付かず、シートにもSlackにも反映されないことを確認

### テスト 3: 完了日時の自動入力
1. シートで任意の行のステータス列（G列）を「完了」に変更
2. K列に当日日時が自動入力されることを確認

### テスト 4: 期限超過の色付け
1. I列に昨日の日付を入力
2. セルが赤色になることを確認
3. ステータスを「完了」に変更すると色が消えることを確認

### 即時実行（5分待たない場合）
1. Apps Script エディタで `fetchAndSync` を選んで「実行」
2. すぐに処理が走る

### スプレッドシートのカスタムメニューから操作（推奨）
スプレッドシートを開き直すと、メニューバーに **「問い合わせパイプライン」** が追加される。Apps Script エディタを開かなくても以下を実行できる：

- **今すぐメール取得**: `fetchAndSync` を即実行
- **Slack通知テスト**: メール送信なしで Slack に1件テスト通知を送る（Webhook URL変更時の確認に便利）
- **ヘルスチェック**: プロパティ／シート／ラベル／トリガー／Slack疎通を一括検証してダイアログ表示
- **セットアップ実行（初回のみ）**: `setup` を実行

## 5. 運用ルール

### ステータス遷移

```
未対応 → 対応中 → 完了
   ↓        ↓
  保留 ← ─ ─┘
```

- 新規受信時は必ず `未対応`
- 着手したら `対応中`
- 顧客返信待ち等で停止する場合は `保留`
- クローズ時は `完了`

### 担当者アサイン
- 受信時は未割当（H列空欄）
- Slack通知を見たメンバーが手挙げ式で H列を更新

### 対応期限
- デフォルトは未設定（空欄）
- 急ぎ案件のみ手動で設定
- ステータスが「完了」「保留」になると色付けは消える

## 6. トラブルシューティング

### Slack に通知が来ない
- Apps Script エディタの「実行数」を開きエラーログを確認
- スクリプトプロパティの `SLACK_WEBHOOK_URL` が正しいか確認
- Webhook URL が無効化されていないか確認
- L列が FALSE のままになっていれば、次回 `fetchAndSync` で自動再送される

### シートに行が追加されない
- Gmail にラベル `問い合わせ-未処理` が付いているか確認
- 付いていなければ → Gmailフィルター条件を見直す
- 付いている → Apps Script エディタで `fetchAndSync` を手動実行してエラー確認

### 重複行が出る
- 通常は M列の Message ID 照合で防止されるはず
- 同じスレッドの新しい返信メールは「新規受信」として扱われる（仕様）

### 個人メールが拾われた
- Gmailフィルター条件が広すぎる
- From条件を絞る、件名キーワードを厳しくする

### トリガーが動いていない
- Apps Script エディタ左の時計アイコン「トリガー」を開く
- `fetchAndSync` が 5分毎、`onSheetEdit` がスプレッドシート編集で登録されているか確認
- なければ Apps Script エディタで `setup` を再実行（既存トリガーは削除して再作成される）

### 緊急停止したい
- Apps Script エディタの「トリガー」で `fetchAndSync` を一時削除
- またはスクリプトプロパティの `SLACK_WEBHOOK_URL` を空にする（シート追記は続くがSlack通知は止まる）

---

## 7. セキュリティ上の注意

- Webhook URL は**コード内にハードコードしない**（スクリプトプロパティに格納する設計）
- スプレッドシートの共有範囲は最小限に絞る
- Apps Script は Gmail の「ラベル付きメールの読み取り＋ラベル変更」しか行わない設計
- 送信・削除権限は付与しない

## 付録: GAS スクリプト本体

スクリプトは別ファイル [Code.gs](Code.gs) に格納してあります。中身をそのまま Apps Script エディタの `Code.gs` に貼り付けてください。

**主な構成（428行）**:

| 関数 | 種別 | 役割 |
|---|---|---|
| `fetchAndSync` | エントリポイント | 5分毎にラベル付きメールを取得→シート追記→Slack通知→ラベル更新 |
| `onSheetEdit` | エントリポイント | シート編集時、ステータス＝完了で完了日時を自動入力 |
| `onOpen` | エントリポイント | スプレッドシート起動時にカスタムメニューを追加 |
| `setup` | 手動実行 | シート初期化＋プルダウン＋条件付き書式＋ラベル＋トリガーを一括セットアップ |
| `healthCheck` | 手動実行 | プロパティ・シート・ラベル・トリガー・Webhook疎通を一括検証 |
| `testNotification` | 手動実行 | メール送信なしでSlackに動作確認用通知を送る |
| `menuFetchAndSync` / `menuTestNotification` / `menuHealthCheck` / `menuSetup` | UI | カスタムメニュー用ラッパー |
| `appendInquiryRow_` / `postToSlack_` / `retryUnnotified_` / `getExistingMessageIds_` / `makePreview_` / `escapeForMrkdwn_` / `logError_` / `applyConditionalFormats_` / `ensureLabel_` / `registerTriggers_` | 内部 | ヘルパー関数（末尾 `_` 付きは private 慣習） |

**コード一切を AI で再生成したい場合**:
このフォルダの [共有用_AI制作プロンプト.md](共有用_AI制作プロンプト.md) を使うと、別のAI（ChatGPT・Claude・Gemini）でも同じスクリプトを再生成できます。
